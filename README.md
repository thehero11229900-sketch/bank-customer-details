# 4. Bank Customer Details System

## 1. Introduction
A banking management system to manage customer accounts, transactions, and loan processing.

## 2. Purpose
To provide secure banking services digitally with KYC compliance.

## 3. Functional Requirements

1. Customer registration with KYC.  
2. Account types: Savings, Current, FD, RD.  
3. View transaction history.  
4. Balance inquiry.  
5. Fund transfer (NEFT, RTGS, IMPS).  
6. Loan processing.  
7. Profile updates with re-verification.  
8. Admin management.  

## 4. Non-Functional Requirements

- Real-time balance updates.  
- OTP authentication.  
- Data encryption.  
- RBI compliance.  
- 99.9% uptime.  

## 5. Acceptance Criteria

- AC-1: Account created within 24h after KYC.  
- AC-2: Real-time balance display.  
- AC-3: Transfer < ₹50,000 → OTP.  
- AC-4: 6-month statement PDF.  
- AC-5: Loan eligibility via credit score.  
- AC-6: Sensitive profile updates need re-verification.  

## 6. Compliance & Auditing

- RBI and KYC compliance.  
- Maintain transaction logs for 10 years.  
- Annual financial audit.
